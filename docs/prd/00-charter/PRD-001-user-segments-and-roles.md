# PRD-001: User Segments and Roles

Status: Draft

## Purpose

Define product-facing user groups and their decision needs so downstream PRDs can use consistent role language.

## Segments

- **Student**: Builds a Decision Profile, explores majors and schools, compares programs, and manages application planning.
- **Parent**: Reviews decision evidence, cost, risk, and progress with appropriate collaboration permissions.
- **Counselor**: Manages a portfolio of students, creates deliverables, sponsors access, and interprets data-backed recommendations.
- **Operator**: Maintains data quality, reviews trust issues, and monitors source freshness and auditability.

## Product Rules

- Role behavior must be described in product terms, not implementation permissions.
- Collaboration PRDs own workspace interaction rules.
- Commercial PRDs own paid entitlement packaging.
- Data asset PRDs own source truth and verification rules.

## Downstream Consumers

- User experience PRDs
- Collaboration PRDs
- Commercial and entitlement PRDs
- Trust operations PRDs
